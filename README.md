# Обратная связь — Мероприятие

Простое приложение для сбора обратной связи от участников и ассистентов мероприятия. Работает полностью в браузере — без сервера и базы данных.

## Файлы

| Файл | Назначение |
|------|-----------|
| `feedback.html` | Форма для участников и ассистентов |
| `admin.html` | Панель девелопера (защищена паролем) |

## Как запустить на GitHub Pages

### 1. Создайте репозиторий

Зайдите на [github.com](https://github.com) → **New repository**  
Назовите, например: `event-feedback`  
Сделайте **Public** → Create repository

### 2. Загрузите файлы

На странице репозитория нажмите **Add file → Upload files**  
Перетащите `feedback.html`, `admin.html`, `README.md` → **Commit changes**

### 3. Включите GitHub Pages

Перейдите в **Settings → Pages**  
В разделе **Branch** выберите `main` и папку `/ (root)` → **Save**

Через 1–2 минуты сайт будет доступен по адресу:
```
https://ВАШ_ЛОГИН.github.io/event-feedback/feedback.html
```

Панель администратора:
```
https://ВАШ_ЛОГИН.github.io/event-feedback/admin.html
```

## Пароль администратора

По умолчанию пароль: **`dev2025`**

Чтобы сменить — откройте `admin.html` и найдите строку:
```js
const DEV_PASSWORD = 'dev2025';
```
Замените на свой пароль и снова загрузите файл на GitHub.

> ⚠️ Данные хранятся в `localStorage` браузера — только на том устройстве, где заполнялась форма. Для сбора данных с разных устройств рекомендуется использовать общий компьютер на мероприятии.

## Как пользоваться

1. Откройте `feedback.html` на устройстве мероприятия
2. Участники и ассистенты заполняют форму
3. После мероприятия откройте `admin.html`, введите пароль
4. Скачайте CSV отдельно для участников и ассистентов
